## What's fixed

- Fix offline transcription in installed builds by resolving bundled sidecar files from the packaged `sidecar` resource directory.
- Fix bundled industry pack lookup in installed builds.
- Fix release publishing so GitHub releases always include the signed updater bundle and `latest.json` metadata needed by in-app update checks.

## Why this release matters

- Installed production builds could show `Transcription engine: stopped` even though dev builds worked.
- Manual update checks could fail because the release was missing the updater metadata and signed update bundle.

## Verification

- Built a production NSIS installer locally with `npm run tauri build`.
- Verified the packaged installer includes `_up_\sidecar\transcribe.py` and bundled sidecar resources.
- Verified the release workflow now builds first, then creates the GitHub release from the generated installer, signed `.nsis.zip`, `.nsis.zip.sig`, and generated `latest.json`.
