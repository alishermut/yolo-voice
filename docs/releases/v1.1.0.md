# YOLO Voice v1.1.0: Product Polish Roadmap

## What's new

- Reworks onboarding into a success-first flow with inline dictation testing, clearer defaults, and a cleaner finish step for first-run setup.
- Adds hands-free voice-activated dictation for the supported Parakeet segmented path, including the new `listening -> recording -> transcribing -> done` pill lifecycle.
- Replaces the old command-mode framing with `Text Actions`, including saved actions, a default action flow, migration from legacy prompts, and an in-app quick picker.
- Adds contextual trust and storage visibility across settings, including local-vs-cloud badges, open-folder actions, and JSON export for transcript history.
- Introduces a persisted `Simple / Advanced` settings experience, hides advanced-only sections by default for new users, and adds one-click dictation presets like `Fastest`, `Best Quality`, `Hands-Free`, and `Coding`.

## Why this release matters

- `v1.1.0` turns the recent product-polish roadmap into one cohesive release instead of a pile of disconnected settings and backend improvements.
- New users now have a much clearer first-run path, while returning users keep access to the full engine, vocabulary, and style controls in `Advanced`.
- The app is easier to trust and easier to operate: hands-free dictation is clearer, text transformations are more productized, and local/cloud behavior is visible in the UI instead of implied.
