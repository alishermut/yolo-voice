# YOLO Voice v1.0.4: Distil-Whisper Production Hotfix

## What's new

- Fixes the Distil-Whisper production runtime lookup so the app can find the bundled Python sidecar after updater installs that place resources under `_up_/sidecar`.
- Stops Distil-Whisper from looping between `preparing` and non-ready states after a failed background prepare. Failed prepares now settle into a stable error state with the underlying message visible in Settings.
- Makes Distil-Whisper GPU switching honest: if the bundled Distil runtime does not have CUDA support, the app now explains that GPU mode is unavailable instead of pretending the switch succeeded and falling back to CPU.
- Adds release verification for the bundled `python-env/python.exe` so future release builds fail fast if the Distil Python runtime is missing from the artifact layout.
- Adds focused Distil runtime logs so support bundles and stderr output show which Python runtime was selected and why a GPU switch was rejected.

## Why this release matters

- This is a production reliability hotfix for Distil-Whisper. It targets the updater/runtime mismatch that could leave some installs using system Python, which then caused errors like `No module named 'torch'`.
- It also removes a confusing GPU UX edge case on Windows where Distil-Whisper would briefly show `preparing` and then silently return to CPU because the bundled runtime only exposed CPU PyTorch.
