# YOLO Voice v1.2.0: Push-to-Talk Reliability and Parakeet English

## What's new

- Fixes the root cause of push-to-talk intermittently stopping until an app restart. Heavy work no longer runs inside the Windows keyboard hook callback, so Windows can no longer silently unhook the listener after a slow start.
- Adds a listener supervisor that detects a dead keyboard hook and reinstalls it, instead of failing silently with no log and no user-facing indicator.
- Fixes a race where a quick tap during a slow cold start could leave recording stuck on, so the next press appeared to do the opposite of what was intended.
- Letter keys can now be bound as the dictation hotkey. Keys the backend cannot represent now show an explicit `unsupported` hint instead of appearing to save and then never firing.
- Adds `Parakeet English` (`parakeet-tdt-v2`) as a selectable offline engine alongside the multilingual v3, with per-variant downloads and warm-up on engine switch.
- Makes settings persistence crash-safe: config, profiles, text actions, and vocabulary are now written atomically, and a corrupt file is backed up rather than silently reset to defaults.
- Adds timeouts to the Distil sidecar so a wedged Python process can no longer freeze Distil features, status polling, or app shutdown.
- Fixes settings races where two quick changes could silently revert each other, and stops API key and model fields from firing a save on every keystroke.

## Why this release matters

- The headline complaint this release addresses is the hotkey "randomly dying" mid-session. That was not flakiness in the key handling, it was the OS removing a hook that took too long to return, which is why only a restart brought it back.
- Config corruption previously wiped hotkeys and API keys silently, because a truncated file fell back to defaults without a word. Writes are now atomic and corrupt files are preserved for recovery.
- A hung sidecar used to take the whole app down with it, including shutdown. It now fails in a bounded, recoverable way.

## Verification

- Passed `cargo check`.
- Passed `cargo test` (92 tests, including new coverage for atomic writes, concurrent write races, and hotkey key-name round-tripping).
- Passed `tsc --noEmit`.
- Passed CI on the merged pull request, including the bundled Python sidecar preparation step.

Note: the push-to-talk changes were not exercised by hand in a running app before this release. Verification for them is automated only. If the hotkey misbehaves after updating, roll back to `v1.1.1` and report it.

## Known gaps

- API keys are still stored in plaintext in `config.json`.
- Model downloads are not integrity-checked, so a truncated download can be marked installed.
- Chord capture in settings can still be interrupted by a re-render, and `Escape` binds itself instead of cancelling capture.
