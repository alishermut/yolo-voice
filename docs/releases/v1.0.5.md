# YOLO Voice v1.0.5: Distil-Whisper CUDA Runtime Fix

## What's new

- Bundles a CUDA-enabled PyTorch runtime for Distil-Whisper on Windows so the packaged Distil sidecar can actually load onto NVIDIA GPUs instead of falling back to CPU-only torch.
- Rebuilds cached sidecar environments whenever the sidecar preparation script changes, which prevents CI and release runners from silently reusing the old CPU-only Distil runtime.
- Verifies the bundled Distil runtime during the release workflow and fails the release if the packaged `python-env` does not expose a CUDA-enabled torch build.
- Switches the Windows release artifact to MSI-based packaging because the CUDA-enabled Distil runtime exceeds NSIS's practical size limit for this app.
- Keeps the earlier production runtime lookup and error-state fixes from `v1.0.4`, so upgraded installs still find the bundled Distil runtime correctly under updater layouts.

## Why this release matters

- This is the release that turns Distil-Whisper GPU mode into a real CUDA path for compatible NVIDIA systems.
- Previous production builds could only run Distil-Whisper on CPU because the bundled Python environment shipped a CPU-only torch build, even though the UI exposed Distil GPU switching.
- The Windows updater now publishes MSI-based updater bundles so the larger CUDA runtime can still be signed and delivered through GitHub releases.
