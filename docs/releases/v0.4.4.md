## What's fixed

- Fix offline transcription in installed builds by resolving bundled sidecar files from the packaged `sidecar` resource directory.
- Fix bundled industry pack lookup in installed builds.
- Enable Tauri updater artifact generation during builds and publish the signed updater bundle plus `latest.json` on every GitHub release.

## Why this release matters

- Installed production builds could show `Transcription engine: stopped` even though dev builds worked.
- Manual update checks could fail because releases were missing the updater metadata and signed update bundle.

## Verification

- Built a production NSIS installer locally with `npm run tauri build`.
- Verified the packaged installer includes `_up_\sidecar\transcribe.py` and bundled sidecar resources.
- Verified the Tauri config now enables `createUpdaterArtifacts`, which is required for generating `.nsis.zip` and `.nsis.zip.sig` in Tauri v2.
- Verified the release workflow builds first, then publishes the installer, signed updater bundle, signature, and generated `latest.json`.
